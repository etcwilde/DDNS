Description

MACChord, like a DNS server, is a name resolution system using the Chord DHT algorithm. The project name originally stems from the idea of using the computer's MAC address as the unique identifier in the hash table; however, that is not human-readable. Instead, using any, unrestricted, unique identifier makes it much easier for humans to interact with. What separates MACChord from simply using a DNS is that it is distributed. This means that there is no central authority monitoring all the nodes, each node will help monitor the other nodes in the system, reducing the load of any given machine, and helping to improve the durability of the system. Furthermore, there can be multiple lookup rings, so names can be reused in other parts of the world.

MACChord is attempting to fix the issue of Dynamic IP addresses. A Dynamic IP changes periodically, making it impossible for clients to connect to a host behind one. It would be like having a telephone number that changed every hour, your friends wouldn't be able to call you because they wouldn't know your number. There are services that provide a way around this, using an external server with a static IP to determine when the public IP of a server with a Dynamic IP has changed, then the external server updates its key-value store, correctly routing the traffic to that server. This puts the load on a single server, and gives all control to a central authority, ultimately forcing the user to hand out some money for the service.
